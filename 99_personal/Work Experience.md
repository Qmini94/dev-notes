<p align="right">박 규 민</p>
(주)유비 (2024.01 ~ 재직 중)

- 직위: 연구원
- 역할: 레거시 CMS 솔루션의 리빌딩과 모던 기술 스택으로의 마이그레이션을 주도하며, 시스템설계 및 개발을 수행합니다. 또한 조직도 도메인의 전반적인 개발 및 운영을 담당합니다.
- 주요 책임: 
	1. 레거시 시스템의 현대화 설계 및 마이그레이션 개발
	2. 조직도 도메인 기능 개발 및 유지보수
	3. 폐쇄망-외부망 보안 데이터 연동 및 ETL 프로세스 수립
- 주목할만한 성과:
	1. 아키텍처 고도화 및 운영 안정성 확보
		- 무중단 배포 파이프라인 구축으로 배포 다운타임 제로 달성
		- 하이브리드 인증 아키텍처 설계로 인증 부하 최소화 및 장애 시 서비스 가용성 확보
		- 도메인 특성에 맞는 Persistence 계층 최적화로 데이터 처리 효율 극대화
	2. 웹 성능 최적화 및 사용자 경험 개선
		- 하이브리드 렌더링 도입으로 Lighthouse 성능 점수 93점(상위 5%) 달성
		- 계층형 상속 권한 조회 병목 해소, 응답 속도 N배 개선
	3. 데이터 처리 시스템 및 기능 확장
		- 검색 엔진 및 비동기 처리 시스템 구축으로 색인 속도, 검색 정확도 동시 향상
		- 런타임 동적 스키마(DDL) 제어 엔진 개발로 관리자 자율 데이터 구조 운영 실현

**[LLM 기반 설계·개발 프로세스 도입 및 워크플로 구축]**
AI 기반 개발 역량 확보가 조직 내 요구되었으나, CLI 환경에서 LLM을 실질적인 설계·개발 워크플로에 적용한 경험자가 없어 주도적으로 프로세스 도입을 진행했습니다. BMad 에이전트 프레임워크와 Claude Code CLI를 활용해 요구사항 정의, 아키텍처 설계, API 계약, 이벤트 흐름을 MD 산출물 중심으로 표준화하고, 에이전트/페르소나/스킬(.skills) 기반 규칙 체계를 수립했습니다.

LLM이 생성한 설계 및 코드 초안은 그대로 채택하지 않고, 도메인 제약 조건, 인터페이스 정합성, 비기능 요구사항(성능·장애·권한) 누락 여부를 점검하는 검증 프로세스를 구축했습니다. 이를 통해 AI 산출물을 단순 보조 도구가 아닌 **검증 가능한 설계·개발 입력물**로 활용하는 체계를 마련했습니다.

그 결과, 설계 산출물 작성 리드타임을 단축하고 인터페이스 변경에 따른 재작업을 감소시켰으며, 설계 의사결정 일관성을 확보하여 신규 기능 확장 시 안정적인 구조 확장을 가능하게 했습니다.

**[CI/CD 파이프라인 구축 및 무중단 배포]**
- Jenkins 기반 CI/CD 파이프라인 구성, 무중단 배포(Blue-Green 또는 Rolling) 적용
- 프론트엔드 public 정적 자원을 빌드 대상에서 분리하여 배포 시간 단축
- 모든 배포 아티팩트가 단일 경로를 참조하도록 통합하여 환경별 경로 불일치 문제 해소

**[Docker 기반 로컬 개발환경 구축 및 최적화]**
- MySQL, Redis, Elasticsearch, Nginx 등 멀티 컨테이너 환경을 Docker Compose로 구성
- Windows 환경에서 bind mount I/O 병목 분석 후 WSL2 네이티브 파일시스템으로 전환하여 빌드 속도 5~10배 개선
- 프론트/백엔드 개발자, 디자이너 간 동일 환경 공유로 "내 PC에서는 되는데" 문제 해소

레거시 CMS 솔루션의 MSA 기반 마련을 위한 모듈러 아키텍처 설계 및 마이그레이션
 복잡한 조회는 QueryDSL로, 대량 데이터 처리는 JDBC로 이원화하여 유지보수성과 성능을 동시에 확보했습니다.
아키텍처 고도화 및 운영 안정성 확보
		- Jenkins 기반의 무중단 배포 파이프라인을 구축하여 배포 시 서비스 중단 시간을 제거하였고, 배포 에러 발생 시 즉각적인 롤백이 가능하도록 운영 안정성 향상시켰습니다.
		- JWT와 Redis를 결합한 하이브리드 인증으로 부하를 최소화하고, 장애 발생 시에도 서비스 가용성을 유지하도록 개선하였습니다.
		- 도메인 특성에 맞춰 JPA, QueryDSL, JDBC를 선별 적용하여 데이터 처리를 최적화하였습니다.
	2. 웹 성능 최적화 및 사용자 경험 개선
		- SSR과 CSR의 하이브리드 렌더링 및 2단계 캐싱 전략(Spring Cache + Static JSON)을 도입하여, 초기 로딩 속도를 개선(Lighthouse 성능 점수 93점)하였습니다.
		- Interceptor 단계에서 SecurityContext 내 JWT Payload를 활용해 권한을 검증하도록 설계했습니다. 특히 Redis 캐싱 단위를 메뉴 노드 단위로 세분화하여, 복잡한 상속 권한 조회 시 발생하는 오버헤드를 제거하고 응답 속도를 극대화했습니다.
	3. 데이터 처리 시스템 및 기능 확장
		- Elasticsearch 도입 및 Redis Stream 기반의 비동기 이벤트 처리로 색인 속도와 검색 정확도 동시 향상시켰습니다.
		- 런타임에 테이블 스키마(DDL)를 제어하는 게시판 시스템을 구축하여 서비스 확장성을 확보하고 관리 효율성을 증가시켰습니다.
