유비 (2024.01 ~ 재직 중)

- 직위: 연구원
- 역할: 레거시 CMS 솔루션의 리빌딩과 모던 기술 스택으로의 마이그레이션을 주도하며, 시스템설계 및 개발을 수행합니다. 또한 조직도 도메인의 전반적인 개발 및 운영을 담당합니다.
- 주요 책임:
	1. 레거시 CMS 시스템의 현대화 설계 및 마이그레이션 개발
	2. 조직도 도메인 기능 개발 및 유지보수
	3. 폐쇄망-외부망 보안 데이터 연동 및 ETL 프로세스 구축
- 주목할만한 성과:
	1. 아키텍처 고도화 및 운영 안정성 확보
		- 무중단 배포 파이프라인 구축
		- 하이브리드 인증 아키텍처 설계로 인증 부하 최소화 및 장애 시 서비스 가용성 확보
		- JPA/QueryDSL/JDBC 선택적 적용으로 도메인별 데이터 처리
	2. 웹 성능 최적화 및 사용자 경험 개선
		- SSR/CSR 하이브리드 렌더링으로 Lighthouse 성능 점수 93점 달성
		- 계층형 상속 권한 조회 쿼리 수 75% 감소 및 응답 속도 10배 이상 개선
	3. 데이터 처리 시스템 및 기능 확장
		- Elasticsearch 도입 및 색인 동기화로 검색 정확도, 데이터 일관성 동시 확보
		- 런타임 동적 스키마(DDL) 제어 구조로 관리자 자율 데이터 구조 운영 실현

**[CI/CD 파이프라인 구축 및 무중단 배포]**
배포시 안정화와 무중단 처리를 위해 백엔드와 프론트엔드 각각에 독립된 Jenkins 파이프라인을 구성하여 빌드, 아카이빙, 배포를 자동화했습니다. CI/CD를 별도 파이프라인으로 분리하여, 빌드 아티팩트를 검증한 후 승인을 거쳐 배포하는 구조를 채택했습니다.

배포에는 Blue/Green 전략으로 무중단 전환하였으며 완료 후 이전 포트의 프로세스를 종료하며, 실패 시 이전 릴리스 디렉토리가 보존되어 즉시 롤백이 가능합니다.

그 결과,  배포 시 사용자에게 서비스 중단 없이 새 버전이 반영되어 운영 안정성과 사용자 경험을 상승시켰습니다.

**[Nuxt 3 SSR/CSR 하이브리드 렌더링 전략]**
SEO와 초기 로딩 속도를 위해 SSR/CSR 하이브리드 전략을 설계했습니다. 개발 및 유지보수시Hydration Mismatch 빈도를 줄이기 위해 미들웨어 파이프라인(IP 접근 제어, 인증, 메뉴 관리, 렌더 권한, 레이아웃 선택)을 분리하여 생산성을 상승시켰습니다.

Static JSON와 Spring Cache로 2단계 캐싱 구조를 설계로 로딩 속도를 개선하여 Lighthouse 점수 93점을 달성하였으며, 백엔드 서버 장애가 있더라도 안정적인 페이지 로딩으로 가용성을 확보하였습니다.

**[동적 게시판 테마 렌더링]**
수십개의 게시판 유형을 관리하는 게시판 유형을 동적으로 렌더링하는 구조를 설계했습니다. 게시판 유형별 테마 컴포넌트(리스트, 검색박스, 페이지네이션)를 조합하여 list, view, write 모드를 동적 렌더링합니다.

이 구조를 통해 신규 게시판 테마 추가 시 테마 디렉토리에 컴포넌트만 추가하면 되고, 라우터나 빌드 설정 변경이 불필요합니다. 백엔드 동적 게시판 CRUD 서비스와 결합하여 코드 변경 없이 관리자 UI에서 새 게시판 유형을 정의하면 즉시 운영할 수 있는 동적 게시판 시스템을 구현했습니다.

**[JWT+Redis 하이브리드 인증]**
강제 로그아웃, 권한 변경 즉시 반영, 슬라이딩 세션 방식 요구사항을 충족시키기 위해 유효한 세션만 Redis에 저장하는 화이트리스트 방식을 채택했습니다. Health Check를 통해 Redis 장애 시에는 Graceful Degradation으로 전환하였습니다.

이를 통해 운영 요구사항을 충족하면서도 Redis 장애 시에도 서비스 가용성을 유지하는 인증 체계를 구축했습니다. 세션 만료 시각을 프론트엔드에 전달하여 클라이언트 측 자동 로그아웃 타이머와 서버 인증 상태 간의 동기화를 확보했습니다.

**[JDBC Template 기반 동적 게시판]**
모든 게시판 유형의 데이터가 하나의 테이블에 집중되는걸 막기 위해 게시판 유형별로 독립 테이블을 런타임에 생성하는 동적 게시판 구조를 설계했습니다. JDBC Template 기반 동적 SQL 빌더를 선택했습니다.

테이블 분리를 통해 게시판별 데이터 규모가 관리 가능한 수준으로 유지되어 단일 테이블 50만 건 LIKE기반 검색과 복합 검색조건 시 병목 해소되었고, 각 테이블에 최적화된 인덱스를 독립 적용할 수 있게 되었습니다.

이 엔진을 통해 신규 게시판 유형 추가 시 코드 변경 없이 관리자 UI에서 필드를 정의하면 즉시 사용 가능한 게시판이 생성됩니다. 프론트엔드 테마 렌더링 엔진과 결합하여 완전한 동적 게시판 시스템을 구현했으며, Elasticsearch 색인과 연계하여 분산된 동적 테이블의 데이터를 통합 검색할 수 있는 구조를 확보했습니다.

**[Redis Streams를 활용한 로그인 AOP에서 CRUD 전체 감사 로깅]**
로그인 성공 및 실패 이력을 포함한 콘텐츠 생성, 수정, 삭제, 권한 변경, 회원 관리 등 모든 CRUD 작업의 감사 추적이 필요했으나, 기존 로깅 유틸리티는 애플리케이션 로그(SLF4J)에만 기록하여 DB 검색이 불가능했습니다. CRUD 감사 이벤트는 수백명의 사용자로도 빈도수가 높아 DB에 직접 쓰기 부하를 줄이기 위해 비동기 Redis Streams로 구현하였습니다.

이를 통해 핵심 트랜잭션 성능에 영향을 줄이면서 사용자의 디테일한 감사 추적을 확보하였습니다.

**[상속 권한 조회 최적화 및 Redis 노드별 캐싱]**
기존 레거시 구조를 개선하기 위해 pathId(dot path) 필드를 도입하여 메뉴의 전체 경로를 문자열(예: "1.3.8")로 기록하여 조상 조회는 문자열 분할 한 번으로, 하위 노드 조회는 LIKE 접두사 쿼리 한 번으로 처리하였습니다. 

Redis 캐싱은 전체 권한을 하나의 키에 담지 않고, 메뉴 노드별로 분리하였습니다. 

그 결과, 재귀 쿼리 제거와 단일 쿼리 조상 조회로 권한 해소 쿼리 수가 75% 감소하고, Redis 노드별 캐싱으로 응답 속도가 20ms 이하로 10배 이상 개선되었습니다. 

[폐쇄망-외부망 보안 데이터 연동 및 ETL 프로세스 구축]
조직, 회계 등 내부 핵심 데이터가 폐쇄망에 격리되어 있어 외부망 서비스에서 활용할 수 없는 구조적 제약이 있었습니다. 폐쇄망 API와 인증서 기반 암복호화 통신으로 데이터를 수신하고, 이기종 시스템 간 스키마 차이를 정제/변환하는 ETL 프로세스를 구축하여 외부망 DB로 연동하는 파이프라인을 설계했습니다.

이를 통해 수동 데이터 이관 작업을 자동화하고, 조직/회계 데이터의 정합성을 유지하면서 외부망 서비스에서 실시간에 가까운 데이터 활용이 가능한 구조를 확보했습니다.

**[LLM 기반 설계/개발 프로세스 도입]**
AI 기반 개발 역량 확보가 조직 내 요구되었으나, CLI 환경에서 LLM을 실질적인 설계/개발 워크플로에 적용한 경험자가 없어 주도적으로 프로세스 도입을 진행했습니다. BMad 에이전트 프레임워크와 Claude Code CLI를 활용해 MD 산출물 중심으로 표준화 하였습니다. 또한, LLM이 생성한 설계 및 코드 초안을 서브 에이전트로 재검증하는 프로세스를 구축했습니다. 

이를통해 MVP개발 기간을 90% 감소시켰으며 설계 의사결정 일관성을 확보하여 신규 기능 확장이나 각 도메인별 고도화 작업 시 효율성을 향상 시켰습니다.

**[Trouble Shooting: Gradle에서 Task와 Process 충돌 빌드 실패 이슈 해결]**
QueryDSL Q클래스 생성 Task와 MapStruct 어노테이션 Process가 동일 라운드에서 실행되며 빌드 실패가 반복되는 문제가 발생했습니다. Lombok, MapStruct, QueryDSL 세 프로세서가 같은 컴파일 단계에서 경합하면서 Q클래스를 참조하지 못하거나 매핑 코드 생성이 실패하는 것이 원인이었습니다.

QueryDSL Q클래스 생성을 -proc:only 옵션으로 별도 JavaCompile Task로 분리하고, compileJava가 이를 의존하도록 실행 순서를 명시했습니다. MapStruct와 Lombok은 메인 컴파일 단계에서만 동작하도록 조건 분기하여 프로세서 간 간섭을 제거했습니다. 또한 디버깅용 classpath 출력 Task를 추가하여 빌드 문제 발생 시 원인 추적이 가능한 구조를 확보했습니다.

이를 통해 빌드 안정성을 확보하고 CI/CD 파이프라인에서의 빌드 실패를 제거했습니다.

[Trouble Shooting: 메뉴 동기화 탐색 병목 현상 해결]
커스텀한 @ExecutionTime으로 메뉴 트리구조 동기화 메서드가 500ms를 초과하는 병목을 발견하였습니다. 이를 해결하기 위해 DFS에서 BFS로 전환하였습니다.

- 동기화시 같은 깊이의 노드를 한 번에 배치 저장하고 pathId를 CONCAT 쿼리로 일괄 갱신
- DB 호출 횟수가 노드 수 비례에서 트리 깊이 비례로 감소

이를 통해 재귀함수 호출 빈도가 60% 이상 감소하였으며 pathId를 갱신이 간소화 되어 코드 가독성을 높였습니다.

[Trouble Shooting: 게시판 도메인 무결성과 통일성 확보]
런타임 동적 게시판 생성 서비스를 구현하며 게시판 관리 테이블, 필드 관리 테이블, 실제 게시판 데이터 테이블의 무결성과 통일성을 확보하였습니다.
- JVM Lock 처리: 해당 솔루션은 단일 애플리케이션으로 운영되기 때문에 JVM Lock 처리하였습니다.
- 보상 롤백 처리: JDBC로 처리하고 있기 때문에 예외시 각 메타/정의 테이블 물리테이블 복구 처리하였습니다.
- 소유자 검증: 물리테이블에 생성에 대한 보상 롤백은 Drop처리이기 때문에 소유자 검증으로 해당 호출에서 만들어진 실제 테이블인지 확인하였습니다.

**[Evidence-based: 적극적인 문서화를 통한 상호 커뮤니케이션 강화]**
다양한 비즈니스 요구조건에 맞는 파이프라인 구현, 프론트 미들웨어, 백엔드 인증 등 철저한 문서화를 통해 업무 효율성과 팀 간 커뮤니케이션을 강화했습니다. 구두 설명으로 인한 비효율을 줄이고, 팀의 생산성이 상승 할 수 있도록 체계적으로 기록하였습니다.

낸드소프트 (2022.03 ~ 2023.01)

- 직위: 연구원
- 역할: 클라이언트의 요청에 맞게 보안 솔루션 기반 개발 및 커스텀마이징
- 주요 책임:
	1. 보안 솔루션 기반 커스텀마이징 개발 
	2. 기존 보안 솔루션 Spring 프레임워크 Spring boot로 마이그레이션. 
- 주목할만한 성과:
	1. 조직도 트리 UI 개발 
	2. Util을 사용하여 리펙토링 및 Test Case 작성 후 검증

**[조직도 트리 UI 개발]**
신입으로 들어가 가장 처음 받은 업무는 따로 분리 되어있는 Read권한만 있는 모듈 인터페이스를 활용하여 컨트롤러에서 호출하여 조직도 트리를 개발하는것이었습니다. 

프론트는 thymeleaf로 서버 렌더링 구조룰 사용하였습니다. 바닐라 스크립트로 초기화 로직과 트리 UI 개발하였으며, 각 이벤트 호출시 상태값들을 초기화하며 전역변수를 최소화하여 개발하였습니다.

백엔드는 각각의 인터페이스를 test case를 given, when, then으로 분리 작성하여 사용하며 검증하였고, REST API로 CRUD를 개발하였으며 Response/Request DTO를 생성하여 각각의 API마다 정의하였습니다.

이를 통해 Spring Boot 마이그레이션 작업에 팀원들과 협업하여 안정적인 솔루션을 만들 수 있었습니다.
