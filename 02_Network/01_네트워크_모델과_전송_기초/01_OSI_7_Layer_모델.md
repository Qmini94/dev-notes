## 1. 개요

OSI 7 Layer 모델은 이기종 시스템 간의 통신 호환성을 보장하고, 네트워크 통신 과정을 계층별로 표준화하기 위해 ISO(국제표준화기구)에서 정의한 참조 모델입니다.

현대 인터넷 인프라는 TCP/IP 4계층 모델을 기반으로 구현되어 있으나, 네트워크 장애 발생 시 문제의 원인을 계층 단위로 격리(Isolation)하고 분석하기 위한 논리적 프레임워크로서 OSI 7 Layer 모델이 표준으로 사용됩니다. 본 문서는 각 계층의 역할과 데이터 처리 흐름, 그리고 실무 환경에서의 구성 요소를 정의합니다.

---

## 2. 데이터 캡슐화 및 전송 흐름

네트워크 통신은 송신 측의 **캡슐화(Encapsulation)**와 수신 측의 **디캡슐화(Decapsulation)** 과정을 통해 이루어집니다.

데이터는 상위 계층에서 하위 계층으로 전달되며, 각 계층을 통과할 때마다 해당 계층의 제어 정보가 담긴 **헤더(Header)**가 데이터 앞부분에 부착됩니다. 반대로 수신 측에서는 하위 계층부터 헤더를 분석하고 제거하며 원본 데이터를 상위 계층으로 전달합니다.

이러한 구조는 각 계층의 독립성을 보장하여, 특정 계층의 변경(예: 랜 케이블 교체)이 다른 계층(예: 웹 브라우저)에 영향을 미치지 않도록 설계되었습니다.

## 3. 계층별 상세 명세

각 계층은 고유한 기능을 수행하며, 상위 계층에 서비스를 제공합니다. 본 장에서는 물리적 인프라 영역(L1~L4)과 논리적 애플리케이션 영역(L5~L7)으로 구분하여 기술합니다.

### 3.1 Physical Layer (1계층, 물리 계층)

데이터 전송의 물리적 기반을 제공하는 최하위 계층입니다. 상위 계층(Data Link)으로부터 전달받은 데이터(Frame)를 통신 매체에 적합한 전기적, 광학적 신호(Signal)로 변환하여 전송합니다.

- **주요 기능**: 비트(Bit) 단위의 신호 전송, 물리적 연결 설정 및 해제.
- **데이터 단위**: Bit
- **실무 구성 요소**:
    - **매체**: UTP 케이블, 광케이블(Fiber Optic).
    - **장비**: 리피터, 더미 허브, 트랜시버(SFP/GBIC).
- **운영 관점**:
    - 서버 및 네트워크 장비 인터페이스의 Link Up/Down 상태를 관리합니다.
    - 물리적 결선 상태 및 케이블의 단선 여부가 통신의 전제 조건이 됩니다.

### 3.2 Data Link Layer (2계층, 데이터 링크 계층)

물리적으로 직접 연결된 인접 노드(Node-to-Node) 간의 신뢰성 있는 데이터 전송을 담당합니다. 물리 계층의 잡음이나 오류를 감지하고 수정하는 기능을 수행하며, **MAC 주소(Media Access Control Address)**를 통해 장비를 식별합니다.

- **주요 기능**: 회선 제어, 흐름 제어, 오류 제어, 프레임 동기화.
- **데이터 단위**: Frame
- **실무 구성 요소**:
    - **식별자**: MAC 주소 (예: `00:0c:29:1a:2b:3c`).
    - **장비**: L2 스위치, 브릿지, NIC(Network Interface Card).
    - **프로토콜**: Ethernet, PPP.
- **운영 관점**:
    - 동일 네트워크 대역(LAN) 내에서의 통신을 담당합니다.
    - 스위칭(Switching) 기술을 통해 MAC 주소 테이블을 기반으로 포트 간 데이터를 전달합니다.
    - 서버의 NIC Bonding(Teaming) 설정 및 VLAN 구성이 이 계층에서 수행됩니다.

### 3.3 Network Layer (3계층, 네트워크 계층)

서로 다른 네트워크 대역에 존재하는 목적지까지 데이터를 전송하기 위한 최적의 경로(Routing)를 결정합니다. 논리적 주소인 **IP 주소**를 사용하여 패킷을 목적지까지 전달합니다.

- **주요 기능**: 라우팅(Routing), 패킷 포워딩, 논리적 주소 지정(Addressing).
- **데이터 단위**: Packet
- **실무 구성 요소**:
    - **식별자**: IP 주소 (IPv4, IPv6).
    - **장비**: 라우터, L3 스위치.
    - **프로토콜**: IP, ICMP, ARP, OSPF, BGP.
- **운영 관점**:
    - End-to-End 통신의 도달성을 보장합니다.
    - `ping` 테스트(ICMP)를 통해 네트워크 경로의 가용성을 진단합니다.
    - 서버의 IP, Subnet Mask, Gateway 설정이 올바르지 않을 경우 외부 네트워크와 통신할 수 없습니다.

### 3.4 Transport Layer (4계층, 전송 계층)

최종 목적지(Host)에 도착한 데이터를 어떤 애플리케이션(Process)으로 전달할지 식별하고, 데이터 전송의 신뢰성을 보장합니다. **포트(Port) 번호**를 통해 서비스를 구분합니다.

- **주요 기능**: 세그먼테이션, 흐름 제어, 오류 제어, 서비스 식별(Port).
- **데이터 단위**: Segment (TCP) / Datagram (UDP)
- **실무 구성 요소**:
    - **식별자**: Port 번호 (예: HTTP 80, HTTPS 443, SSH 22).
    - **장비**: L4 스위치(로드 밸런서), 방화벽.
    - **프로토콜**: TCP(신뢰성 보장), UDP(비연결형 고속 전송).
- **운영 관점**:
    - 서버 내 특정 프로세스의 데몬(Daemon) 활성화 여부를 판단합니다.
    - 방화벽 정책(Inbound/Outbound) 및 로드 밸런싱 설정의 기준이 됩니다.
    - `telnet [IP] [Port]` 명령을 통해 포트 리스닝 상태 및 방화벽 차단 여부를 점검합니다.

### 3.5 Session Layer (5계층, 세션 계층)

응용 프로그램 간의 논리적인 연결(Session)을 생성, 관리, 종료하는 계층입니다. 통신 장치 간의 상호 작용을 설정하고 동기화합니다.

- **주요 기능**: 세션 설정 및 해제, 동기화, 대화 제어(Duplex).
- **데이터 단위**: Message / Data
- **실무 구성 요소**:
    - **프로토콜**: RPC, NetBIOS, SSH, TLS Handshake(세션 수립 단계).
- **운영 관점**:
    - TCP/IP 세션의 생명주기를 관리합니다.
    - 애플리케이션 레벨의 Keep-Alive 설정이나 DB Connection Pool 관리 메커니즘이 이 계층의 개념을 포함합니다.

### 3.6 Presentation Layer (6계층, 표현 계층)

응용 계층으로부터 전달받은 데이터의 형식을 네트워크가 이해할 수 있는 표준 형식으로 변환하거나, 그 반대의 작업을 수행합니다. 서로 다른 시스템 간의 구문(Syntax) 차이를 해결합니다.

- **주요 기능**: 데이터 인코딩/디코딩, 암호화/복호화, 압축/해제.
- **데이터 단위**: Message / Data
- **실무 구성 요소**:
    - **포맷**: ASCII, JPEG, MPEG.
    - **보안**: SSL/TLS (데이터 암호화).
- **운영 관점**:
    - HTTPS 통신 시 인증서 기반의 암호화 및 복호화가 수행되는 단계입니다.
    - JSON, XML 등의 데이터 직렬화(Serialization) 과정이 포함됩니다.

### 3.7 Application Layer (7계층, 응용 계층)

최상위 계층으로, 사용자가 네트워크 서비스에 접근할 수 있는 인터페이스를 제공합니다. 사용자 입력을 받아 하위 계층으로 데이터를 전달하거나, 수신된 데이터를 사용자에게 표현합니다.

- **주요 기능**: 프로세스 간 통신, 사용자 인터페이스 제공, 네트워크 서비스 수행.
- **데이터 단위**: Message / Data
- **실무 구성 요소**:
    - **장비**: L7 스위치(ADC), WAF(웹 방화벽).
    - **프로토콜**: HTTP/HTTPS, DNS, FTP, SMTP, SSH.
- **운영 관점**:
    - 실질적인 비즈니스 로직과 애플리케이션 코드가 동작하는 영역입니다.
    - HTTP Status Code(404, 500 등)를 통해 애플리케이션의 상태를 진단합니다.

## 4. 계층 기반 장애 진단 프로세스

네트워크 및 서비스 장애 발생 시, OSI 7 Layer 모델을 기반으로 하위 계층부터 상위 계층으로(Bottom-Up) 원인을 분석하여 문제 범위를 축소합니다.

|진단 순서|점검 계층|확인 내용|주요 점검 항목|
|:--|:--|:--|:--|
|**Step 1**|**L1 Physical**|물리적 연결|케이블 결선, 인터페이스 Link LED 점등 여부|
|**Step 2**|**L2 Data Link**|로컬 통신|NIC 상태, 스위치 포트 상태, MAC 주소 학습 여부|
|**Step 3**|**L3 Network**|네트워크 도달성|IP/Gateway 설정, `ping` 테스트 (Packet Loss 확인)|
|**Step 4**|**L4 Transport**|서비스 접근성|방화벽 정책, 포트 리스닝(`netstat`), `telnet` 접속 테스트|
|**Step 5**|**L7 Application**|서비스 동작|웹 서버 로그, 애플리케이션 로직 오류, HTTP 응답 코드|

**진단 예시: Ping은 정상이나 웹 접속이 불가한 경우**

1. **L3 (Ping 정상)**: 네트워크 경로 및 서버 전원은 정상임.
2. **L4 점검**: 웹 포트(80/443)로 `telnet` 시도.
    - 접속 실패 시: 방화벽 차단 또는 웹 서버 데몬이 구동되지 않음.
    - 접속 성공 시: 네트워크 및 전송 계층은 정상.
3. **L7 점검**: 애플리케이션 로그 확인. 소스 코드 오류 또는 설정 파일(Context Path 등) 문제로 판단.
