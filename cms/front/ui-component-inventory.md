# UI 컴포넌트 인벤토리

이 문서는 프로젝트에서 사용되는 **UI 컴포넌트 목록과 역할**을 정리합니다.  
각 컴포넌트의 책임과 사용 범위를 명확히 하여  
**재사용성, 일관성, 유지보수성**을 확보하는 것을 목표로 합니다.

본 문서는 개발자와 AI 에이전트 모두가  
**동일한 기준으로 컴포넌트를 선택·조합**하도록 돕습니다.

---

## 컴포넌트 분류 기준

UI 컴포넌트는 다음 기준에 따라 분류됩니다.

- **Base 컴포넌트**: 가장 기본적인 UI 요소
- **Composite 컴포넌트**: 여러 Base 컴포넌트를 조합한 구성 요소
- **Page-Specific 컴포넌트**: 특정 페이지에 종속된 컴포넌트

---

## Base 컴포넌트

### Button

- **설명**: 공통 버튼 컴포넌트
- **역할**: 사용자 액션 트리거
- **규칙**:
  - 스타일은 props로 제어
  - 비즈니스 로직 포함 금지

---

### Input

- **설명**: 공통 입력 필드 컴포넌트
- **역할**: 사용자 입력 수집
- **규칙**:
  - 검증 로직은 외부에서 주입
  - 폼 상태를 직접 관리하지 않음

---

## Composite 컴포넌트

### Modal

- **설명**: 공통 모달 컴포넌트
- **역할**: 사용자 확인/입력 흐름 제공
- **구성**:
  - Header
  - Body
  - Footer

---

### Table

- **설명**: 데이터 목록 표시 컴포넌트
- **역할**: 리스트/테이블 렌더링
- **규칙**:
  - 데이터 소스는 외부에서 주입
  - 페이징/정렬은 옵션으로 제어

---

## Page-Specific 컴포넌트

### BoardList

- **설명**: 게시판 목록 화면 전용 컴포넌트
- **역할**: 게시글 목록 렌더링
- **주의**:
  - 다른 페이지에서 재사용 금지
  - 페이지 구조 변경 시 함께 수정

---

### BoardForm

- **설명**: 게시글 작성/수정 전용 컴포넌트
- **역할**: 게시글 입력 UI 제공

---

## 재사용 규칙

컴포넌트를 재사용하기 전, 다음 질문에 답해야 합니다.

- 이 컴포넌트는 특정 도메인에 종속적인가?
- 다른 화면에서도 동일한 의미로 사용될 수 있는가?
- props만으로 동작을 제어할 수 있는가?

위 질문 중 하나라도 **아니오**라면,  
Page-Specific 컴포넌트로 분류합니다.

---

## 금지 사항

- 컴포넌트 내부에서 전역 상태(store) 직접 접근
- API 호출 포함
- 권한/인증 로직 포함

이러한 로직은 반드시 상위 계층에서 처리해야 합니다.
