# 프로젝트 개요

이 문서는 본 프로젝트의 전반적인 목적, 범위, 핵심 개념을 설명합니다.  
프로젝트에 참여하는 모든 구성원(기획, 개발, 운영, AI 에이전트 포함)이  
**동일한 맥락과 목표를 공유**할 수 있도록 작성되었습니다.

## 프로젝트 목적

본 프로젝트의 주요 목적은 다음과 같습니다.

- 일관된 구조와 명확한 책임 분리를 통해 유지보수 가능한 시스템 구축
- 기획·설계·개발·운영 전 단계에서 **사고의 연속성(Context Continuity)** 확보
- 인간과 AI가 협업할 수 있는 명확한 기준 문서(md) 제공
- 확장성과 변경 가능성을 고려한 아키텍처 기반 마련

본 프로젝트는 단순히 기능을 구현하는 것을 목표로 하지 않으며,  
**장기적인 운영과 진화를 전제로 한 시스템**을 지향합니다.

## 범위(Scope)

이 프로젝트는 다음 범위를 포함합니다.

- 프론트엔드 애플리케이션 구조
- 데이터 모델 정의
- API 계약 및 통신 규약
- 상태 관리 및 사용자 컨텍스트 처리
- 개발 및 배포 프로세스 문서화

다음 항목은 범위에 포함되지 않습니다.

- 특정 인프라 벤더에 종속적인 구현
- 단기 이벤트성 기능
- 문서화되지 않은 암묵적 규칙

## 핵심 원칙

프로젝트 전반에 적용되는 핵심 원칙은 다음과 같습니다.

### 1. 명시적 설계 (Explicit Design)

모든 중요한 결정은 문서로 남깁니다.  
암묵적인 지식이나 개인 의존적인 판단을 최소화합니다.

### 2. 일관성 우선 (Consistency First)

- 코드 구조
- 네이밍
- 데이터 흐름
- API 계약

위 요소들은 일관성을 최우선으로 설계합니다.

### 3. 변경 가능성 전제 (Designed for Change)

요구사항 변경은 피할 수 없는 전제입니다.  
따라서 구조는 변경을 수용할 수 있도록 설계되어야 합니다.

### 4. 인간 + AI 협업

본 프로젝트는 AI를 자동화 도구가 아닌 **협업 파트너**로 활용합니다.

- 문서는 AI에게도 읽히는 1급 자산입니다.
- 설계와 의사결정의 맥락을 문서로 고정합니다.
- AI는 이 문서를 기반으로 일관된 결과를 도출합니다.

## 문서 사용 가이드

- 이 문서는 **단독으로 읽기보다**,  
  `architecture.md`와 함께 읽는 것을 권장합니다.
- 프로젝트 방향성에 대한 논의가 필요할 경우,  
  반드시 본 문서를 기준점으로 삼습니다.
- 본 문서의 변경은 프로젝트 전반에 영향을 미칠 수 있으므로  
  변경 시 명확한 합의와 이력이 필요합니다.
