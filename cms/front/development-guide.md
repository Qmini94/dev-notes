이 문서는 프로젝트의 **로컬 개발 환경 구성, 개발 규칙, 작업 흐름**을 정의합니다.  
모든 개발자는 동일한 기준과 절차를 따르는 것을 전제로 합니다.

본 가이드는 인간 개발자뿐 아니라  
AI 기반 개발 워크플로우에서도 **일관된 결과를 보장**하기 위한 기준 문서입니다.

---

## 개발 환경 요구 사항

- Node.js (LTS 버전 권장)
- 패키지 매니저: npm 또는 pnpm
- Git

모든 개발자는 동일한 런타임 환경에서 작업해야 하며,  
환경 차이로 인한 이슈를 최소화합니다.

---

## 프로젝트 실행

```bash
npm install
npm run dev
```

- 기본 개발 서버는 로컬 환경에서 실행됩니다.
    
- 환경 변수는 `.env` 파일을 통해 관리합니다.

## 개발 규칙

### 코드 스타일

- ESLint / Prettier 설정을 따릅니다.
    
- 자동 포맷팅 결과를 신뢰합니다.
    
- 임의의 스타일 규칙 추가 금지
    

---

### 네이밍 규칙

- 변수, 함수, 파일명은 **의미 중심**으로 작성
    
- 약어 사용 최소화
    
- 도메인 개념은 문서(data-models.md)와 일치해야 함
    

---

### 상태 관리 규칙

- 전역 상태는 stores/에서만 관리
    
- 컴포넌트에서 직접 상태 변경 금지
    
- 상태 변경은 액션을 통해서만 수행
    

---

### API 사용 규칙

- 모든 API 호출은 API 레이어를 통해 수행
    
- 컴포넌트에서 직접 fetch 호출 금지
    
- API 계약(api-contracts.md)을 기준으로 구현
    

---

## 작업 흐름

1. 요구사항 확인 (문서 기준)
    
2. 영향 범위 파악
    
3. 구현
    
4. 테스트
    
5. 문서 업데이트 (필요 시)
    

문서 없는 구현은  
**완료된 작업으로 간주하지 않습니다.**

---

## 테스트 정책

- 주요 로직은 테스트 가능 구조로 설계
    
- 테스트 코드 없는 핵심 로직 추가 지양
    

---

## AI 협업 가이드

- AI에게 작업을 요청할 때는 반드시 관련 문서를 함께 제공
    
- 문서와 다른 결과가 나오면 문서를 우선 기준으로 판단
    
- 임의 추론 결과를 그대로 반영하지 않음
