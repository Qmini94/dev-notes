# 프로젝트 문서 인덱스

이 문서는 본 프로젝트의 전체 문서 구조에 대한 진입점입니다.  
프로젝트의 목적, 아키텍처, 데이터 모델, API 계약, 개발 및 배포 가이드 등  
모든 핵심 문서를 체계적으로 탐색할 수 있도록 구성되어 있습니다.

## 문서 구조

아래 문서들은 프로젝트의 생명주기 전반을 다루며,  
기획 → 설계 → 구현 → 운영 흐름에 맞춰 정리되어 있습니다.

- **[프로젝트 개요](project-overview.md)**  
  프로젝트의 목적, 범위, 핵심 목표 및 전체적인 방향성을 설명합니다.

- **[아키텍처](architecture.md)**  
  시스템의 전체 구조, 주요 컴포넌트, 책임 분리, 설계 원칙을 다룹니다.

- **[데이터 모델](data-models.md)**  
  주요 엔티티, 관계, 데이터 구조 및 제약 조건을 정의합니다.

- **[API 계약](api-contracts.md)**  
  클라이언트와 서버 간의 인터페이스, 요청/응답 구조, 에러 규약을 명시합니다.

- **[소스 트리 분석](source-tree-analysis.md)**  
  프로젝트 디렉토리 구조와 각 영역의 역할을 설명합니다.

- **[UI 컴포넌트 인벤토리](ui-component-inventory.md)**  
  재사용 가능한 UI 컴포넌트 목록과 사용 목적을 정리합니다.

- **[개발 가이드](development-guide.md)**  
  로컬 개발 환경 설정, 개발 규칙, 워크플로우를 설명합니다.

- **[배포 가이드](deployment-guide.md)**  
  빌드, 배포, 환경 구성 및 운영 관련 절차를 안내합니다.

- **[기여 가이드](contribution-guide.md)**  
  프로젝트에 기여하는 방법, PR 규칙, 커뮤니케이션 가이드를 포함합니다.

## 사용 대상

이 문서는 다음과 같은 대상을 위해 작성되었습니다.

- 프로젝트에 새로 참여하는 개발자
- 기획 및 설계를 이해해야 하는 팀원
- AI 기반 개발 워크플로우(BMad 등)에 컨텍스트를 제공해야 하는 경우
- 장기적인 유지보수 및 확장을 고려하는 이해관계자

각 문서는 독립적으로 읽을 수 있지만,  
**index → overview → architecture** 순서로 읽는 것을 권장합니다.
